# grill-boss

把领导/老板的一句话模糊需求，拷问成可执行、可落地的方案。

设计参考 [Matt Pocock 的 batch-grill-me]：需求是一棵设计树，按**前沿（frontier）分轮批量提问**——每轮只问前置已定的问题，逐题编号并附推荐答案；事实自己查（可后台并行），决策留给人；前沿清空才交付。

在此基础上针对"领导需求"场景做了三处扩展：

1. **三向分诊** —— 问题分为：自己查的事实、问用户的决策、只有领导能拍板的决策。
2. **问领导清单** —— 问领导的机会昂贵，压缩到 ≤5 条封闭式问题，每条附建议选项 + 一句话理由，领导 10 秒点头即可；附"没空细看则按建议推进"的兜底。
3. **假设台账** —— 问不到的决策不许悄悄写进方案：登记默认值、依据、猜错代价，随方案交付；方案中每个数字（含日期、工期）要么有查证来源、要么在台账里。

## 安装

复制 `SKILL.md` 到个人技能目录：

```
~/.claude/skills/grill-boss/SKILL.md
```

或作为项目技能放在项目的 `.claude/skills/grill-boss/SKILL.md`。

## 使用

对 Claude 说"领导说要搞X，帮我弄一下"之类的话会自动触发；也可显式说"grill boss"或"拷问需求"。

## 测试记录（TDD for skills）

按 superpowers:writing-skills 的 RED-GREEN-REFACTOR 流程开发：

- **RED（无技能基线）**：两个模糊领导需求场景。失败模式：直接产出带编造数字的通用方案、一次性倒出所有问题不管依赖、事实与决策不分流、无问领导清单、共识未成就要动手。
- **GREEN（加载技能）**：两场景均正确完成三向分诊、前沿分轮编号提问带推荐、≤5 条封闭式问领导清单带兜底、承诺假设台账、未提前产出方案。
- **REFACTOR**：发现代理编造"今天周四"未核实日期 → 补充"日期/工期也算数字，须查系统时间或进台账" → 重测确认代理改为用工具查证系统日期。
- **v2 合并**（吸收另一版 grill-with-boss 的设计）：中间人框架（领导不在对话里）、第一轮打树根"揣摩上意"、假设替代请示的降级规则、【待补】占位符与"初稿"话术红线、"急"只压缩轮次预算（一轮 3–5 题，其余转显式假设）。重测："别问了直接写、初稿也行"的双重压力场景下代理拒绝初稿、压缩为一轮 5 题；场景 A 回归确认第一轮以意图层打头。
